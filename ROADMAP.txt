  FAZ 1 — DİSASSEMBLY & KOD GEZGİNİ            Öncelik: ★★★★★
══════════════════════════════════════════════════════════════════════════

  Amaç: Entry point ötesinde tam binary navigasyonu

  [ ] 1.1  Tam disassembly görünümü
           — EP dışında herhangi bir offset'e atlama
           — Sonsuz scroll (yukarı/aşağı, lazy-load chunk'lar)
           — Adres çubuğu: RVA/VA/dosya ofseti gir → atla
           — Syntax renklendirme: CALL (mavi), JMP (turuncu), NOP (gri),
             RET (kırmızı), veri referansları (yeşil)

  [ ] 1.2  Fonksiyon listesi paneli
           — Rust tarafında CALL/RET talimatlarından fonksiyon sınırları çıkar
           — Sol panelde fonksiyon adı + boyut + çağrı sayısı listesi
           — Tıkla → disassembly o fonksiyona atla
           — Import ismiyle eşleştirme (IAT cross-reference)

  [ ] 1.3  Temel blok grafiği (CFG)
           — JMP/Jcc/CALL/RET'ten basic block ayrıştır
           — React Flow veya D3.js ile interaktif grafik
           — Blok tıkla → disassembly'de vurgula
           — Otomatik layout (dagre/elk algoritması)

  [ ] 1.4  Cross-reference (XRef) tablosu
           — Bir fonksiyona/string'e kimler referans veriyor?
           — String çift tık → XRef listesi
           — Import fonksiyonu çift tık → çağıran talimatları göster

  [ ] 1.5  Disassembly içinden hex patch
           — Talimat sağ tık → "NOP'la", "JMP ile değiştir"
           — Değişiklik anında Patcher kuyruğuna eklensin

  FAZ 2 — AI GELİŞMİŞ ZEKA                     Öncelik: ★★★★☆
══════════════════════════════════════════════════════════════════════════

  Amaç: AI'ı pasif sohbetten aktif analiz partnerine dönüştür

  [ ] 2.1  Bağlamsal bellek (Context Memory)
           — Son N tarama özetini otomatik system prompt'a ekle
           — "Bu binary'yi daha önce tarattım, hatırlıyor musun?"
           — Bellek yönetimi: temizle / max token limiti ayarla

  [ ] 2.2  Çoklu dosya AI karşılaştırması
           — İki binary'nin scanner JSON'larını yan yana AI'a gönder
           — "Bu ikisi arasındaki farkları analiz et" tek tıkla

  [ ] 2.3  Hex aralığı açıklama
           — Hex viewer'da bölge seç → "AI'a Sor" butonu
           — Seçili baytlar + offset bilgisiyle AI'a gönder
           — "Bu 64 byte ne olabilir? Struct olarak tahmin et"

  [ ] 2.4  Otomatik IOC çıkarımı
           — AI yanıtlarından IP/domain/hash/yol otomatik parse et
           — Bulunan IOC'ları ayrı tabloda göster
           — Tek tıkla YARA kuralına dönüştür

  [ ] 2.5  AI güven skoru
           — Her yanıta %0-100 kesinlik derecesi ekle
           — "Emin değilim" vs "Yüksek olasılıkla" renk kodlu

  [ ] 2.6  Fonksiyon decompile tahmini
           — Disassembly'den seçili fonksiyonu AI'a gönder
           — "Bu assembly'yi C/C++ pseudocode'a çevir"
           — Sonucu syntax-highlighted panel'de göster

  [ ] 2.7  Otomatik YARA kural sihirbazı
           — Scanner bulgularından AI destekli kural üretimi
           — Kullanıcı düzenleyebilsin, test edebilsin
           — Kural kütüphanesi (kaydet/yükle/paylaş)

  FAZ 3 — RUST BACKEND GÜÇLENDİRME             Öncelik: ★★★★☆
══════════════════════════════════════════════════════════════════════════

  Amaç: Frontend yükünü azalt, performansı artır

  [ ] 3.1  Tam PE parsing'i Rust'a taşı
           — JS'de yapılan string çıkarımı, YARA, heuristic → Rust'a al
           — ~500ms JS parse → <50ms Rust parse

  [ ] 3.2  UPX dışında packer unpack desteği
           — ASPack, PECompact, MPRESS, Petite algılama + unpack
           — Generic entropy-based unpack denemesi

  [ ] 3.3  Memory dump analizi geliştirme
           — .dmp dosya formatı proper parsing
           — Process memory region'ları ayırt etme
           — PE reconstruction from dump

  [ ] 3.4  ssdeep fuzzy hashing
           — Benzer dosya tespiti için ssdeep entegrasyonu
           — Geçmişteki taramalarla benzerlik skoru

  [ ] 3.5  Paralel toplu tarama
           — Rust thread pool ile çoklu dosya eşzamanlı analiz
           — İlerleme çubuğu: dosya X/N, %tahmini süre

  [ ] 3.6  ELF / Mach-O desteği (opsiyonel)
           — goblin zaten ELF + Mach-O destekliyor
           — Scanner'ı PE-spesifik olmaktan çıkar, multi-format yap

  TEKNİK NOTLAR
══════════════════════════════════════════════════════════════════════════

  • Disassembly:  capstone-rs zaten entegre. Rust tarafında chunk-based
    disassembly endpoint'i eklemek yeterli. Frontend'de sanal scroll
    (react-window veya custom virtualizer) gerekli.

  • CFG:  react-flow veya elkjs layout ile. Rust'tan basic block
    JSON döndür, frontend'de render et. ~500 talimata kadar sorunsuz.

  • AI Bellek:  Basit yaklaşım — son 3 tarama özetini system prompt'a
    JSON olarak ekle. Token limiti aşarsa en eski özeti düşür.

  • PDF:  jsPDF + html2canvas veya Tauri print komutu. Rapor şablonunu
    HTML olarak oluştur, CSS @media print ile tasarla.

  • i18n:  react-i18next veya basit bir JSON sözlük sistemi. Tüm
    hardcoded Türkçe stringleri anahtar-değer çiftine taşı.

  • Plugin Sandbox:  Web Worker veya <iframe sandbox> ile izole et.
    Plugin'e sınırlı API objesi geç (window erişimi yok).

  • Performans:  String çıkarımı + YARA + heuristic kontrolleri
    Rust'a taşınırsa tarama süresi %80 düşer.
    Mevcut ~4100 satırlık App.jsx'i component'lara bölmek
    bakım kolaylığı için kritik (Scanner, Patcher, Chat, System
    ayrı dosyalar).

  • Kod Organizasyonu Önerisi:
    src/
      components/
        Scanner/     — ScannerPage + alt bileşenler
        Patcher/     — PatcherPage + hex viewer
        Chat/        — ChatPage + mesaj bileşenleri
        System/      — SystemPage + GPU wizard
        Plugins/     — PluginPage + sandbox
        shared/      — Card, Modal, Tooltip, vs.
      hooks/         — useScanner, usePatcher, useChat, useTheme
      utils/         — pe-parser, yara-engine, formatters
      i18n/          — tr.json, en.json
      App.jsx        — sadece routing + layout (~200 satır)
