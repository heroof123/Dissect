  FAZ 8 — GERÇEK BACKEND ENTEGRASYONLARı        Öncelik: ★★★★★
══════════════════════════════════════════════════════════════════════════

  Amaç: FAZ 6'daki mock/simüle sayfaları gerçek Rust
        backend'lere bağla. Windows API entegrasyonu.

  [ ] 8.1  Gerçek Process Enumeration (Rust)
           — windows-rs crate ile EnumProcesses / OpenProcess
           — Süreç listesi gerçek PID/isim/modules/threads
           — Yetki kontrolü (IsElevated check)
           — Tauri komutu: list_processes

  [ ] 8.2  Gerçek bellek okuma (Rust)
           — ReadProcessMemory API entegrasyonu
           — VirtualQueryEx ile bellek bölge haritası
           — Tauri komutu: read_process_memory(pid, addr, size)
           — Tauri komutu: query_memory_regions(pid)

  [ ] 8.3  Gerçek debugger (Rust)
           — windows-rs: DebugActiveProcess / WaitForDebugEvent
           — SetBreakpoint (INT3 injection)
           — GetThreadContext / SetThreadContext (registers)
           — Single-step (TRAP flag)
           — Tauri komutları: attach_debugger, set_breakpoint,
             step_into, continue_execution, get_registers

  [ ] 8.4  Gerçek disassembly-on-attach
           — Attach edilen sürecin belleğini oku → capstone ile disassemble
           — RIP/EIP'den itibaren canlı disassembly
           — Tauri komutu: disassemble_memory(pid, addr, size)

  [ ] 8.5  Unicorn Engine emülasyon (Rust)
           — unicorn-engine crate entegrasyonu
           — Seçili fonksiyonu emüle et
           — Register snapshot her adımda
           — Bellek yazma callback'leri
           — Tauri komutu: emulate_function(bytes, arch, start_addr)

  [ ] 8.6  Ağ izleme (Rust)
           — ETW (Event Tracing for Windows) ile process network events
           — veya raw socket / npcap entegrasyonu
           — DNS query, TCP connect, HTTP request capture
           — Tauri komutu: start_network_capture(pid), stop_capture

══════════════════════════════════════════════════════════════════════════
  TEKNİK NOTLAR
══════════════════════════════════════════════════════════════════════════

  • Kod Bölme:  Vite dynamic import + React.lazy ile. Her Page
    component'ı kendi dosyasında. Shared state → Context veya Zustand.
    Migrasyon sırası: en bağımsız sayfadan başla (Dashboard → System
    → Plugins → Patcher → Chat → Scanner → Disasm → App shell).

  • windows-rs:  Process enumeration + debugger için windows crate.
    Cargo.toml'a features: ["Win32_System_Diagnostics_Debug",
    "Win32_System_Threading", "Win32_System_ProcessStatus",
    "Win32_Security"]. Admin yetki: manifest'e requestedExecutionLevel.

  • Unicorn Engine:  unicorn-engine Rust binding mevcut. x86/x64 arch,
    hook_add ile instruction/memory callback. Emulation → JSON state
    snapshot dizisi olarak frontend'e gönder.

  • RAG:  sqlite-vss (SQLite vector search) veya tantivy (Rust FTS).
    Her tarama sonucunu embedding'e dönüştür (local model ile).
    Soru geldiğinde → benzer taramaları bul → context'e ekle.

  • AI Agent:  Tauri komutlarını sırasıyla çağıran bir loop. Her
    adımda AI'a "şimdiye kadarki bulgularım" + "sıradaki hamle ne?"
    sor. ReAct format: Thought → Action → Observation → repeat.

  • Cross-platform:  Tauri 2 zaten Linux/macOS destekler. Platform-
    specific kodlar (windows-rs) → cfg(target_os) ile koşullu compile.
    Linux'ta ptrace, macOS'ta mach_* syscalls.

  • CLI:  clap crate + ayrı bin target (src/bin/dissect_cli.rs).
    JSON output → stdout. Non-interactive batch mode.
